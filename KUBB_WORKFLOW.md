# Kubb Client Generation Workflow

This document describes how the Kubb code generation pipeline works in WineMarket.

## Overview

```
Backend (Elysia + Zod)
  тЖУ Routes with Zod schemas
OpenAPI Spec (auto-generated by @elysiajs/openapi)
  тЖУ Available at http://localhost:3000/swagger/json
Kubb Code Generation
  тЖУ Reads OpenAPI, generates typed clients
Frontend Hooks (@repo/api)
  тЖУ Typed React Query hooks ready to use
React Components
  тЖУ Fully type-safe API integration
```

## Prerequisites

Before running Kubb, ensure:

1. **Server is running:** `bun run server:dev`
   - Starts on port 3000
   - OpenAPI spec generated at `/swagger/json`

2. **Dependencies installed:** `bun install`
   - Kubb packages in `apps/web`
   - @tanstack/react-query for hooks

## Step-by-Step Workflow

### Step 1: Define Backend Routes

In `apps/server/src/modules/<module>/<module>.routes.ts`:

```typescript
import { Elysia } from 'elysia'
import { z } from 'zod'

// 1. Define Zod schema (this is your source of truth)
const ProductSchema = z.object({
  id: z.string().uuid(),
  name: z.string().min(1),
  price: z.number().positive(),
})

// 2. Create routes with Zod validation
export const productRoutes = new Elysia({ prefix: '/api/products', tags: ['products'] })
  .get(
    '/',
    ({ query }) => {
      // Fetch all products
      return []
    },
    {
      query: z.object({ category: z.string().optional() }),
      response: z.array(ProductSchema),
    }
  )
  .post(
    '/',
    ({ body }) => {
      // Create product
      return { id: 'uuid', ...body }
    },
    {
      body: ProductSchema.omit({ id: true }),
      response: ProductSchema,
    }
  )
```

### Step 2: Check OpenAPI Spec

Start the server and navigate to the OpenAPI UI:

```bash
bun run server:dev
# Visit: http://localhost:3000/swagger
# Or fetch JSON: http://localhost:3000/swagger/json
```

The spec should show all your routes with correct request/response schemas.

### Step 3: Generate Kubb Hooks

From the project root:

```bash
bun run generate
```

This executes `kubb generate` in `apps/web`:

```bash
cd apps/web && kubb generate
```

Kubb reads the OpenAPI spec and generates:

```
apps/web/src/generated/
тФЬтФАтФА clients/          # Fetch clients (one-time requests)
тФВ   тФЬтФАтФА getProducts.ts
тФВ   тФЬтФАтФА postProducts.ts
тФВ   тФФтФАтФА ...
тФФтФАтФА hooks/            # React Query hooks (cached, reactive)
    тФЬтФАтФА useGetProducts.ts
    тФЬтФАтФА usePostProducts.ts
    тФФтФАтФА ...
```

### Step 4: Use Generated Hooks in Components

```typescript
// apps/web/src/routes/products.tsx
import { useGetProducts, usePostProducts } from '@/generated/hooks'
import { useQueryClient } from '@tanstack/react-query'

export function ProductsPage() {
  const { data: products, isPending } = useGetProducts()
  const createMutation = usePostProducts()
  const queryClient = useQueryClient()

  const handleCreateProduct = async (formData: any) => {
    await createMutation.mutateAsync(formData)
    // Refetch products after creation
    queryClient.invalidateQueries({ queryKey: ['products'] })
  }

  if (isPending) return <div>Loading...</div>

  return (
    <div>
      <h1>Products</h1>
      {products?.map(p => (
        <div key={p.id}>
          <h3>{p.name}</h3>
          <p>${p.price}</p>
        </div>
      ))}
      <button onClick={() => handleCreateProduct({ name: 'New Product', price: 29.99 })}>
        Add Product
      </button>
    </div>
  )
}
```

## Workflow Integration

### Before Committing Backend Changes

```bash
# 1. Make changes to backend routes/schemas
# 2. Test server locally
bun run server:dev

# 3. Regenerate types
bun run generate

# 4. Update frontend to use new hooks
# 5. Test frontend
bun run dev

# 6. Commit
git add .
git commit -m "feat: add product categories endpoint"
```

### CI/CD Pipeline

In the CI pipeline, this workflow is:

```yaml
1. Install dependencies
2. Run lint (ESLint + Biome)
3. Run type-check (TypeScript)
4. Build (both frontend and backend)
```

Note: Code generation doesn't run in CI; it should happen locally before pushing.

## Troubleshooting

### "OpenAPI spec not found"

```
Error: Cannot fetch http://localhost:3000/swagger/json
```

**Solution:** Start the server first:
```bash
bun run server:dev
```

### "Generated types don't match backend"

**Solution:** Regenerate after backend changes:
```bash
bun run generate
```

### "useGetProducts hook not found"

```
Cannot find module '@/generated/hooks/useGetProducts'
```

**Cause:** Hook wasn't generated (backend route name mismatch).

**Solution:**
1. Check backend route name matches expected hook name
2. Routes with names like `GET /api/products/:id` generate hooks like `useGetProductsById`
3. Regenerate: `bun run generate`

## Key Principles

- **Zod schemas are the source of truth** тАФ they drive OpenAPI generation
- **OpenAPI spec is auto-generated** тАФ never edit it manually
- **Generated code is read-only** тАФ don't edit `src/generated/`
- **Regenerate after backend changes** тАФ every new route/schema change
- **TypeScript is your safety net** тАФ if types mismatch, TypeScript will error
