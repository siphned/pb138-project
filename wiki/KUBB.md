# Kubb — Code Generation from OpenAPI

## What is Kubb?

**Generates typed API clients and React hooks** from an OpenAPI specification.

```
Elysia Routes
    ↓
openapi.json (generated by @elysiajs/openapi)
    ↓
kubb.config.ts (your config)
    ↓
Generated Code:
  - Typed fetch clients (getProducts, postProducts, etc.)
  - React Query hooks (useGetProducts, usePostProducts, etc.)
  - Zod validators
```

Backend changes → TypeScript yells at you. No runtime surprises.

---

## The Problem It Solves

### ❌ Without Kubb

Manual fetch code, duplicated everywhere:

```ts
// page-1.tsx
const response = await fetch('/api/products')
const products = await response.json()
// What's the shape? Who knows. No types.

// page-2.tsx
const response = await fetch('/api/products', {
  method: 'POST',
  body: JSON.stringify({ name: 'Widget', price: 9.99 })
})
// Did I get the params right? Was price required?
// Is category optional or required?
```

**Problems:**
- No type safety
- No autocomplete
- Easy to typo endpoints
- Backend changes break silently

### ✅ With Kubb

Generated, typed hooks:

```ts
import { useGetProducts } from '@/generated/hooks/useGetProducts'
import { usePostProducts } from '@/generated/hooks/usePostProducts'

const { data, isPending } = useGetProducts()
//    ^^^^ { id, name, price, category }[] — fully typed!

const mutation = usePostProducts()
mutation.mutate({ 
  name: 'Widget', 
  price: 9.99,
  category: 'electronics'  // if you forget this → TS error
})
```

**Benefits:**
- Type safety end-to-end
- Autocomplete on request/response shapes
- No manual fetch code
- Backend changes caught immediately

## Setup

```bash
bun add -D @kubb/cli @kubb/core @kubb/plugin-client @kubb/plugin-react-query
bun add @tanstack/react-query
```

```ts
// apps/web/kubb.config.ts
import { defineConfig } from '@kubb/core'
import { pluginClient } from '@kubb/plugin-client'
import { pluginReactQuery } from '@kubb/plugin-react-query'

export default defineConfig({
  input: {
    path: 'http://localhost:3000/swagger/json', // from Elysia @elysiajs/openapi
  },
  output: {
    path: 'src/generated',
  },
  plugins: [
    pluginClient({
      output: { path: 'clients' },
    }),
    pluginReactQuery({
      output: { path: 'hooks' },
    }),
  ],
})
```

Scripts already configured in workspace `package.json`:
```json
{
  "scripts": {
    "generate": "turbo run generate"
  }
}
```

Run from project root whenever the backend changes:
```bash
bun run generate
```

This will regenerate hooks in `apps/web/src/generated/` based on latest OpenAPI spec.

## Generated Artifacts

### Fetch Clients

```ts
// src/generated/clients/getProducts.ts
export const getProducts = async (options?: RequestInit) => {
  const response = await fetch('/api/products', options)
  return response.json()
}
```

**Usage:**
```ts
import { getProducts } from '@/generated/clients/getProducts'

const products = await getProducts()
```

Typed fetch function — one-time network request, no caching.

### React Query Hooks

```ts
// src/generated/hooks/useGetProducts.ts
export const useGetProducts = (
  options?: Omit<UseQueryOptions<...>, 'queryKey' | 'queryFn'>
) => {
  return useQuery({
    queryKey: ['products'],
    queryFn: () => getProducts(),
    ...options
  })
}
```

**Usage:**
```ts
import { useGetProducts } from '@/generated/hooks/useGetProducts'

const { data, isPending, isError } = useGetProducts()
// data: Product[] | undefined
// Full React Query capabilities: caching, background refetch, retry, etc.
```

### Mutations

For POST, PUT, PATCH, DELETE:

```ts
import { usePostProducts } from '@/generated/hooks/usePostProducts'
import { useDeleteProductsById } from '@/generated/hooks/useDeleteProductsById'

// Create
const createMutation = usePostProducts()
createMutation.mutate({ name: 'Widget', price: 9.99, category: 'electronics' })

// Delete
const deleteMutation = useDeleteProductsById()
deleteMutation.mutate({ id: '123' })
```

## Real-World Example

### Backend (Elysia + Zod)

```ts
// modules/products.ts
const CreateProductSchema = z.object({
  name: z.string().min(1).max(100),
  price: z.number().positive(),
  category: z.enum(['electronics', 'clothing', 'food']),
})

const ProductSchema = z.object({
  id: z.string().uuid(),
  ...CreateProductSchema.shape,
})

const products = new Elysia({ prefix: '/products' })
  .get('/', () => {
    // Returns: Product[]
    return getProducts()
  })
  .post('/', ({ body }) => {
    // body: CreateProduct (from schema validation)
    const product = createProduct(body)
    set.status = 201
    return product  // Product
  }, {
    body: CreateProductSchema,
  })
  .get('/:id', ({ params: { id } }) => {
    // Returns: Product | null
    return getProductById(id)
  })
```

### OpenAPI Spec (Auto-Generated)

The `@elysiajs/openapi` plugin automatically generates `openapi.json`:

```json
{
  "openapi": "3.0.0",
  "paths": {
    "/api/products": {
      "get": {
        "summary": "Get all products",
        "responses": {
          "200": {
            "content": {
              "application/json": {
                "schema": {
                  "type": "array",
                  "items": { "$ref": "#/components/schemas/Product" }
                }
              }
            }
          }
        }
      },
      "post": {
        "requestBody": {
          "content": {
            "application/json": {
              "schema": { "$ref": "#/components/schemas/CreateProduct" }
            }
          }
        },
        "responses": {
          "201": {
            "content": {
              "application/json": {
                "schema": { "$ref": "#/components/schemas/Product" }
              }
            }
          }
        }
      }
    }
  },
  "components": {
    "schemas": {
      "Product": { ... },
      "CreateProduct": { ... }
    }
  }
}
```

### Frontend (React + Kubb)

```bash
npm run generate  # Reads http://localhost:3000/openapi.json, generates code
```

```tsx
import { useGetProducts } from '@/generated/hooks/useGetProducts'
import { usePostProducts } from '@/generated/hooks/usePostProducts'
import { useGetProductsById } from '@/generated/hooks/useGetProductsById'
import { ProductSchema } from '@/generated/schemas/Product'

function ProductsPage() {
  // Read
  const { data: products, isPending } = useGetProducts()
  
  // Create
  const createMutation = usePostProducts()
  const handleCreate = (formData: FormData) => {
    createMutation.mutate({
      name: formData.get('name') as string,
      price: Number(formData.get('price')),
      category: formData.get('category') as any,
      // ^ If you typo 'categroy' or forget a required field → TS error
    })
  }

  return (
    <div>
      {isPending && <p>Loading...</p>}
      <ul>
        {products?.map((p) => (
          <li key={p.id}>
            {p.name} — ${p.price} ({p.category})
          </li>
        ))}
      </ul>
      <form onSubmit={(e) => {
        e.preventDefault()
        handleCreate(new FormData(e.currentTarget))
      }}>
        {/* form fields */}
      </form>
    </div>
  )
}
```

**What you get:**
- `products` is typed as `Product[] | undefined`
- `createMutation.mutate()` requires `{ name, price, category }`
- Typos caught at compile time
- Backend changes (new required field) → immediate TS error

## Workflow

1. **Backend:** Define routes with Zod schemas (Elysia)
2. **OpenAPI:** `@elysiajs/openapi` auto-generates `openapi.json`
3. **Generate:** `npm run generate` (Kubb reads the spec)
4. **Frontend:** Import and use generated hooks

**Repeat whenever backend changes:**
```bash
npm run generate
```

## Why Kubb?

| Approach | Pros | Cons |
|---|---|---|
| **Manual fetch** | Full control | No types, boilerplate, easy to break |
| **GraphQL** | Great DX, typed | More complex, requires backend support |
| **Kubb + OpenAPI** | Types, no boilerplate, REST | Requires @elysiajs/openapi |

Kubb bridges the gap — **REST simplicity with GraphQL-like DX**.

## Advanced: Invalidation on Mutation

After creating/updating/deleting, invalidate React Query cache to refetch:

```tsx
import { useQueryClient } from '@tanstack/react-query'
import { usePostProducts } from '@/generated/hooks/usePostProducts'

function CreateProductForm() {
  const queryClient = useQueryClient()
  const createMutation = usePostProducts()

  const handleSubmit = (data: CreateProductInput) => {
    createMutation.mutate(data, {
      onSuccess: () => {
        // After POST succeeds, refetch the product list
        queryClient.invalidateQueries({ queryKey: ['products'] })
      },
    })
  }

  return <form onSubmit={...}>...</form>
}
```

Now when you create a product, the list automatically refetches.
